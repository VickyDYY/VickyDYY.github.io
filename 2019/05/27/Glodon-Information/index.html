广联达相关信息（写在面试前）

Posted on 2019-05-27 | In 面试公司信息

今天晚上18:00就要去面试广联达的产品经理暑期实习生了，在面试前，自然不免需要了解一番公司的情况，以免人家问起公司的情况一脸懵逼。实不相瞒，这是我自4月份找产品实习以来第一次正经的大公司产品面试，处女面就要这样子给出去了555……也不知道自己会表现得怎么样。

先在这里给自己毒奶一波吧！希望不要群面就挂掉……

一、基本情况

广联达科技股份有限公司成立于1998年，2010年5月在A股上市，目前总市值283.98亿。

广联达立足建筑产业，围绕工程项目的全生命周期，是提供以建设工程领域专业应用为核心基础支撑，以产业大数据、产业新金融等为增值服务的数字建筑平台服务商。

广联达现拥有员工五千二百余人，在中国境内建立五十余家分子公司，先后在美国、英国、芬兰、瑞典、新加坡、香港、马来西亚、印度尼西亚、印度等地设立了子公司、办事处与研发中心，服务客户遍布全球一百多个国家。

现在，广联达正在为实现每一个工程项目都接水、接电、接数字建筑平台的二次创业理想而努力，在这一过程中，广联达将作为建筑产业转型升级的核心引擎，助力“中国建造”建立全球核心竞争力！

二、平台理念

品牌使命：追求全体员工的物质和精神幸福，用科技创造美好的生活和工作环境

品牌愿景：成为员工幸福、行业领先、基业长青、世人尊敬的美好企业

价值主张：让每一个工程项目成功Making Every Project A Success

品牌定位：数字建筑平台服务商

发展方针：专业化、平台化、全球化、生态互促

三、发展历程

广联达经过近二十年的发展，公司产品已从单一的预算软件扩展到工程造价、工程施工、工程信息、国际化、产业金融等多个业务板块的近百款产品，覆盖项目全生命周期，涵盖工具类、解决方案类、电子商务、大数据、移动互联网、云、智能硬件设备、产业金融服务等业务形态，累计为行业二十余万家企业、百余万产品使用者提供专业化服务，成果中国建筑产业信息化卓越品牌。

这家创立于1998年的公司，创业前15年都在做一件事情：把工程造价电算化在全国普及。这15年，它专注于服务一类人，就是工程造价人员。（ps：创业目标：让每一个预算员甩掉计算器）
2013年之后，广联达从点对点的功能软件提供商，开始向建设领域的互联网平台商迈进。涉足到的领域很广，包括造价、大数据、征信、金融等等，开发出了70多款应用。

四、业务板块

广联达主要包含以下四个业务板块：

1、造价业务板块：国内外造价、电子政务、工程信息业务
2、施工业务板块：BIM建造、智慧工地、数字企业
3、创新业务板块：规建管一体化、甲方BIM+PM
4、生态业务板块：新金融、教育培训

其中，施工业务布局包括”两横两纵”：

两横：智慧工地、BIM5D

两纵：进度管理、安全管理

五、核心产品

1、云计价产品：

1.1 产品概述：

广联达云计价平台GCCP5.0是迎合数字建筑平台服务商战略转型，为计价客户群，提供 概算、预算、结算 阶段的数据编制、审核、积累、分析和挖掘再利用的平台产品，于2015年研发完成并投放市场。

平台基于大数据、云计算等信息技术，实现计价全业务一体化，全流程覆盖，从而使造价工作更高效、更智能。

1.2 产品优势：

云计价立足于互联网+建设行业大数据，结合实际业务应用场景，对工程建设行业、大数据、互联网、云计算、IT技术、人工智能等多重强专业性集合的深入研究，充分发挥大数据，云计算的实际应用价值。与传统工程计价软件相比，数据化，云端化的计价软件具有无可比拟的优势：

  • 免部署安装，上网就能用
  • 全业务结合，各业务阶段数据零损耗流转
  • 大数据时代，通过数据复用、智能组价等提高工作效率，通过云检查使工程更准确安全
  • 在线实时分享：信息时代，数据共享已成趋势
  • 多人分工协作：云计价以项目为核心，从任务开始到工程结束，任何存档资料都统一在云端存储，同一个项目、单位工程可以同时分配给不同的人员进行编制，系统自动进行汇总

2、BIM5D

2.1 产品概述：

广联达BIM5D为工程项目提供一个可视化、可量化的协同管理平台。

通过 轻量化 的BIM应用方案，达到减少施工变更、缩短工期、控制成本、提升质量的目的，同时为项目和企业提供数据支撑，实现项目精细化管理和企业集约化经营。

（PS：轻量化：通过分析目标用户最高频的需求，给他们提供最大程度上简化的软件）
2.2 产品定位：

核心服务对象是施工总包单位，核心服务的业务是施工管理

（BIM5D不是一款建模软件，它不能像Revit那样创建BIM模型；它也不是造价软件，不能完成BIM算量这样的工作）

2.3 产品功能：

项目的合同、质量、进度、成本、物料这些信息，关联到已有的BIM模型上，实现这些信息更高效的管理。

2.4 实现逻辑：

广联达建议的BIM实施是这样一个三层的逻辑：

  • 首先，先是在「岗位级」这个层面，比如PC端的成本管理，手机端的任务列表，对于每个具体的使用者，做的还是原来的事，只是换了工具。用手机实时发送现场情况，接收指令，总比开会、写报告要方便的多。这一步，先不谈BIM重应用，让基层人员都能感觉到新工具带来的效率提升。
    而像模型深化和维护这样的工作，广联达则是建议在现阶段不加重原来人员的负担，增设新的BIM部门和岗位，专人专职。
  • 下一步，这些技术人员持续的工作，会把可分析的信息，源源不断提供给项目管理者，让他们更清晰的掌握项目整体情况——这些信息在以前也有，只不过是离散、滞后和低效的。有了这些信息，项目管理者能做出更好的决策和安排，就有了「项目级」的BIM应用。
  • 再下一步，这些数据经过多个项目的实施和积累，最终会成为整个公司的数据库，包括资金走向、利润分析、成本控制优化等等，它们被软件 和云加工汇总，成为「企业级」BIM实施的基础。

3、智慧工地

3.1 建筑工地痛点：
  • 工地分散、安全事故、实时监控难以实现、
  • 数据管理复杂、人员管理难度大、信息流通屏障
3.2 智慧工地需求分析：
  • 实名制管理信息需求
  • 安全管控需求
  • 质量管理需求
  • 绿色环保施工需求
  • 进度管理需求
  • 成本管理需求
3.3 产品概述：

广联达智慧工地平台将施工现场的应用和硬件设备集成到一个统一的平台，并将产生的数据汇集，形成数据中心。

基于智慧工地平台，各个应用之间可以实现数据的互联互通并形成联动，同时平台将关键指标、数据以及分析结果以项目BI的方式集中呈现给项目管理者，并智能识别问题进行预警，从而实现施工现场数字化、在线化、智能化的综合管理，让每一个工程项目成功！

3.4 产品组成：
  • 生产管理模块
  • 物料验收模块
  • 质量安全管理模块
  • 经营管理模块
  • BIM建造模块
  • 项目概况模块
3.5 产品分层
  • 终端层：物联网设备：智能安全帽、人脸识别、安全带、烟火承重检测
  • 平台层：云计算平台，实现信息互通和交互
  • 应用层：PM项目管理系统+BIM可视化、参数化、数据化
